About This Project

A deep dive into network security, cryptography, and systems programming - building an HTTPS server from the ground up using C and OpenSSL.

📚 Project Overview

This HTTPS server is a fully functional web server built entirely in C, demonstrating a comprehensive understanding of network programming, the HTTP protocol, and modern cryptographic security. What you're viewing right now is being served over an encrypted TLS 1.3 connection, with every byte of data protected by industry-standard encryption.

This isn't just a learning exercise—it's a production-quality implementation that handles real HTTPS traffic, complete with certificate authentication, secure key exchange, and encrypted data transmission. The server was built without frameworks or high-level abstractions, providing intimate knowledge of what happens under the hood of every HTTPS request.

⚙️ Technical Implementation

The TLS Handshake

Every HTTPS connection begins with a TLS handshake. Here's what happens when you visit this page:

1. Client Hello

Your browser sends a ClientHello message listing supported TLS versions and cipher suites.

2. Server Hello + Certificate

Server responds with chosen protocol (TLS 1.3), cipher suite (AES-256-GCM), and presents its certificate.

3. Key Exchange

Client and server perform ECDHE key exchange to establish a shared secret without ever transmitting it.

4. Session Keys Derived

Both sides independently compute identical session keys from the shared secret.

5. Encrypted Communication

All subsequent data is encrypted with AES-256-GCM using the derived session keys.

🛡️ Security Features

This server implements multiple layers of security to protect data in transit:

AES-256-GCM Authenticated encryption
TLS 1.3 Latest protocol
ECDHE Perfect forward secrecy
4096-bit RSA Strong key size
SHA-384 Secure hashing
X.509 Certs Identity verification

Why These Matter

Perfect Forward Secrecy (PFS): Even if someone records all our encrypted traffic today and later steals the server's private key, they still can't decrypt past communications. Each session uses ephemeral keys that are destroyed after use.

Authenticated Encryption (GCM): Not only is data encrypted, but any tampering is detected immediately. An attacker can't modify encrypted data without the change being detected.

Modern Ciphers Only: The server refuses to use weak or deprecated ciphers, ensuring all connections use strong cryptography.

🎯 Challenges Overcome

Certificate Path Nightmares

Spent hours debugging why certificates couldn't be found, only to discover the issue was relative paths vs. working directories. Lesson learned: always verify your working directory and use debug logging!

The "No Shared Cipher" Mystery

SSL_accept() kept failing with cryptic errors about cipher suites. Solution required understanding how TLS cipher negotiation works and properly configuring OpenSSL's cipher list for both TLS 1.2 and 1.3.

Memory Leaks Everywhere

Every SSL connection leaked memory because I forgot to call SSL_free(). Valgrind became my best friend for finding and fixing all memory leaks. Clean shutdown is harder than it looks!

HTTP Parsing Edge Cases

Handling malformed requests, missing headers, partial reads, and buffer boundaries required careful string manipulation and bounds checking. One wrong pointer arithmetic and boom—segmentation fault!
